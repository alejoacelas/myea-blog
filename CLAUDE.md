# myea.blog

Single-page index of writing drafts, cross-posts, and bullet-point notes. Each entry links to a Google Doc. Live at `myea.blog`.

## Stack

- Plain HTML, CSS, and JS served from `public/`; Vercel functions live in `api/`.
- `public/index.html` is the homepage; `public/style.css` and `public/index.js` drive its UI.
- `public/llms.txt` and `public/all.txt` are generated by `scripts/build_llms_txt.py` — never edit them by hand.
- Analytics: `public/analytics.js` beacons page views and post clicks to `api/hit.js`; `/stats` reads counts via `api/counts.js`.

## Posts

Each post is an `<li>` in the `.post-list` of `public/index.html`:

- `data-kind` is exactly one of `draft`, `bullet points`, or `cross-post` — `bullet points` for list-shaped notes, `cross-post` for writing mainly published elsewhere, `draft` otherwise. No other values.
- `data-created-time` and `data-modified-time` record the Doc's Drive timestamps; sort order uses `modifiedTime`.
- The `<a class="post-link" href="/<slug>" data-doc="<doc-id>">` carries a clean local slug and the stable Google Doc id. `<span class="post-title">` holds the title; `<span class="post-tag">` holds the bare kind word (the surrounding `[ ]` are added by CSS — don't type them).

Rules for the fields:

- **Title:** the Doc's real title (`gdoc info <doc-id>`), sentence-cased, preserving proper nouns and acronyms such as AI, EA, EAG, EV, Robin, Toulouse, and Claude.
- **Slug + redirect:** every slug needs a matching temporary redirect to the Doc in `vercel.json`, so click analytics keep using the stable Doc id.
- **Order:** newest `modifiedTime` first, with two exceptions for inserted entries (`bullet points` and `cross-post`): they may never be the topmost post, and at most two may sit between two `draft`s. Nudge them out of strict time order to satisfy this, and don't publish one that can't sit between real posts.

## Safety checks (before publishing anything)

- **Jojo / Robin guard:** never publish a post whose title contains `Jojo`, never publish Jojo's own writing, and scan each Doc body for `Jojo` (the public pseudonym is `Robin`). `build_llms_txt.py` aborts if it finds `Jojo`.
- **Ownership:** confirm Alejo owns the Doc — include `'me' in owners` in the Drive query.
- **Link access:** the Doc must be shared `anyoneWithLink`, role `commenter` (`gog drive permissions <id> -j --results-only`). If it isn't, flag it — `gog drive share` cannot grant commenter access.

## Add or update posts

1. Find owned Docs: `gog drive search --raw-query "... and 'me' in owners and mimeType = 'application/vnd.google-apps.document'"`.
2. Drop non-posts: calls, meetings, transcripts, summaries, invoices, applications, untitled docs, contact lists, and already-linked Docs.
3. Read survivors with `gdoc cat <id> --no-images` and classify each (see Posts).
4. Run the safety checks.
5. In `public/index.html`, add or update each `<li>` — refresh `data-modified-time` for every linked Doc, then reorder newest-first and apply the inserted-entry rules. Add or update the slug's redirect in `vercel.json`.
6. Deploy (below).

A Doc that only changed (no new post) still needs steps 5–6: refresh its `data-modified-time`, re-sort, and redeploy so `all.txt` and the order stay current.

## Slugs come from the Doc header

`scripts/add_doc_headers.py` writes `Find this post at myea.blog/<slug>` into each Doc's page header, linked to the live URL. That header is the canonical slug: if a Doc's header slug differs from `public/index.html`, trust the header — update the homepage link and `vercel.json`, and redirect the old slug to the new one. Never break a live URL: any time a served route or slug changes, add a permanent redirect in `vercel.json`.

## Deploy

Vercel project `2026-04-10-my-ea-blog` (org `alejandros-projects-a115cc74`), production domain `myea.blog`.

1. `uv run scripts/build_llms_txt.py` — regenerate `llms.txt` and `all.txt` from `index.html`.
2. Verify locally.
3. `vercel deploy --prod` from the repo root.
4. `uv run scripts/add_doc_headers.py` — set each Doc's `myea.blog/<slug>` header (run once the `/<slug>` redirects are live).
5. Commit.

Both scripts read every linked Doc with bounded concurrency (8 workers, `--workers N`). `add_doc_headers.py` also takes `--dry-run` and `--only <slug>`.
